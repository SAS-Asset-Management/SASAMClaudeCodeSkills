Push Notifications Skill v1.0.0

Send notifications to Microsoft Teams and macOS desktop when tasks complete, errors occur, or attention is needed. Two channels, five notification types, full integration with other SASAMClaudeCodeSkills.

Teams Webhooks

Send Adaptive Card messages to any Microsoft Teams channel via incoming webhook URL.

Desktop Alerts

Native macOS notifications via terminal-notifier with sound and click actions.

Skill Integration

Automatically notify when other skills complete — beam-selling, tender-assessment, and more.

How It Works

The notification workflow follows seven steps from command to confirmation:

  1. Parse Command — Determine if setup, test, send, or channel-specific
  2. Load Config — Read .notifications/config.json or trigger setup
  3. Validate Channels — Check enabled channels are still functional
  4. Format Message — Apply notification type styling (icon, colour)
  5. Send Notification — Execute curl (Teams) and/or terminal-notifier (desktop)
  6. Update Statistics — Increment counters in config
  7. Report Result — Confirm success or explain failure

Teams Channel

Send messages to a Microsoft Teams channel via incoming webhook. Messages are formatted as Adaptive Cards with type-specific styling.

Webhook Integration

Use the Bash tool to send via curl. The payload uses the Adaptive Card format:

curl -H "Content-Type: application/json" -d @- "WEBHOOK_URL" << 'EOF'
{
  "type": "message",
  "attachments": [{
    "contentType": "application/vnd.microsoft.card.adaptive",
    "content": {
      "$schema": "http://adaptivecards.io/schemas/adaptive-card.json",
      "type": "AdaptiveCard",
      "version": "1.4",
      "body": [
        {
          "type": "TextBlock",
          "text": "✓ {{TITLE}}",
          "weight": "Bolder",
          "size": "Medium",
          "color": "Good"
        },
        {
          "type": "TextBlock",
          "text": "{{MESSAGE}}",
          "wrap": true
        },
        {
          "type": "FactSet",
          "facts": [
            {"title": "Skill", "value": "{{SKILL_NAME}}"},
            {"title": "Time", "value": "{{TIMESTAMP}}"}
          ]
        }
      ]
    }
  }]
}
EOF

Desktop Channel

Setup Process

When no configuration exists, the skill runs a first-time setup interview covering four areas:

1. Notification Channels

Which notification channels would you like to enable?

  • Microsoft Teams
  • Desktop (macOS)
  • Both

2. Microsoft Teams Setup

If Teams is selected, check whether the user has an incoming webhook URL ready. If not, provide the setup guide from references/setup-guide-teams.md.

3. Desktop Setup

Check if terminal-notifier is installed via which terminal-notifier. If not installed, provide the installation guide from references/setup-guide-desktop.md.

4. Default Preferences

Configure default preferences:

  • Should notifications include sound? (default: yes)
  • Default notification type? (default: info)

Config File

Configuration is stored in .notifications/config.json. This file is created during setup and updated after each notification.

  "schema_version": "1.0.0",
  "created_at": "2026-02-25T10:30:00Z",
  "updated_at": "2026-02-25T10:30:00Z",

Notification Workflow

The full notification lifecycle from trigger to confirmation:

Discovery

Check if .notifications/config.json exists. If not, run first-time setup. Validate webhook URL and terminal-notifier availability.

Delivery

Format the message with type-specific icon and colour. Send via curl (Teams) and/or terminal-notifier (desktop). Update statistics.

Error Handling

Teams Webhook Unreachable

If the webhook endpoint cannot be reached, the skill will attempt desktop notification as a fallback (if configured), log the error, and advise the user to check network connectivity or re-run /push-notifications setup teams to reconfigure.

terminal-notifier Not Installed

If terminal-notifier is not found in PATH, the skill will fall back to Teams (if configured), skip desktop notifications, and suggest installing via brew install terminal-notifier (macOS) or sudo apt install libnotify-bin (Linux).

No Channels Configured

If no notification channels are configured, the skill cannot send any notifications. Run /push-notifications setup to configure Microsoft Teams (incoming webhook) and/or Desktop notifications (terminal-notifier).

Security

Webhook URL Protection

  • Store webhook URL in .notifications/config.json (should be in .gitignore)
  • Never display the full webhook URL — mask it: https://outlook.office.com/webhook/...abc123
  • Validate URL starts with https:// before saving

HTTPS Validation

All webhook URLs must use HTTPS. The skill will reject any HTTP URLs during setup and warn the user to provide a secure endpoint.

Testing Webhook

When the user provides a webhook URL, the skill sends a test message to verify it works before saving to config. This confirms the URL is valid and the channel is reachable.

Content Guidelines

Message Formatting Rules

  • Keep titles under 50 characters
  • Keep messages under 200 characters
  • Be specific: "Tender assessment complete — 3 shortlisted" not "Task done"
  • Include context: which skill, what happened, what to do next
  • Use Australian English spelling throughout

Pre-Delivery Checklist

Before Sending

  • Configuration loaded from .notifications/config.json
  • At least one channel is configured and enabled
  • Message is not empty
  • Message type is valid (success, error, warning, info, attention)
  • Teams webhook URL is HTTPS (if Teams enabled)
  • terminal-notifier exists in PATH (if desktop enabled on macOS)

After Sending

  • Update statistics in config (total_sent, channel counts, last_sent_at)
  • Report success or failure to user
