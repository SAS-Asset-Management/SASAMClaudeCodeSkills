Website Analytics v1.0.0

Expert guidance for the SAS-AM Website Analytics system. Collects data from Google Analytics 4 and Google Search Console, stores it in SQLite, and visualises it through Docker-based Grafana dashboards.

GA4 Integration

User behaviour, traffic sources, engagement metrics, and session data from Google Analytics 4.

Search Console

Keyword rankings, click-through rates, impressions, and SERP position tracking.

SQLite Storage

~160k rows across 4 tables with full query access via CLI or Grafana.

Grafana Dashboards

5 interactive dashboards running in Docker at localhost:8000.

System Architecture

Google APIs Python Scripts SQLite Database (GA4 + Console) → (Data Collection) → (website_analytics.db) ↓ Grafana Docker Container Dashboards ← (localhost:8000) ←────────┘

Data flows from Google APIs through Python collection scripts into a local SQLite database. Grafana runs in a Docker container, reading directly from the SQLite file via the frser-sqlite-datasource plugin.

Database Schema

The SQLite database (website_analytics.db) contains four data tables plus a metadata table.

console_data (~120k rows)

Google Search Console metrics for keyword and page performance.

ColumnTypeDescription
DateTEXTDate of the search data
PageTEXTURL of the page
QueryTEXTSearch query that triggered the impression
DeviceTEXTDevice type (DESKTOP, MOBILE, TABLET)
CountryTEXTCountry code
ClicksINTEGERNumber of clicks from search
ImpressionsINTEGERNumber of search impressions
CTRREALClick-through rate
PositionREALAverage search position

user_data (~21k rows)

Google Analytics user behaviour and traffic sources.

ColumnTypeDescription
DateTEXTDate of the session data
Source, Medium, CampaignTEXTTraffic source attribution
Country, Device CategoryTEXTVisitor segments
Total Users, New Users, SessionsINTEGERUser and session counts
Engagement Rate, Bounce RateREALEngagement metrics (0–1 scale)
Average Session Duration (secs)REALTime on site

page_data (~11k rows)

Page-level performance metrics.

ColumnTypeDescription
DateTEXTDate of the page data
Page Title, Page PathTEXTPage identification
ReferrerTEXTTraffic source for page
Page Views, SessionsINTEGERView and session counts
Engagement TimeREALTime spent on page

website_data (~7k rows)

Consolidated analytics combining all sources into a unified view.

Grafana Query Formats

The frser-sqlite-datasource plugin requires specific query formatting. Use timeColumns: ["time"] in the target configuration.

Time Series

SELECT
  strftime('%Y-%m-%dT%H:%M:%SZ', Date) as time,
  SUM("Page Views") as "Page Views"
FROM page_data
WHERE Date >= date('now', '-${period}')
GROUP BY DATE(Date)
ORDER BY time;

Stat Panel

SELECT SUM("Total Users") as value
FROM user_data
WHERE Date >= date('now', '-${period}');

Table Query

SELECT Country, SUM("Total Users") as Users
FROM user_data
WHERE Date >= date('now', '-30 days')
GROUP BY Country
ORDER BY Users DESC
LIMIT 10;

Troubleshooting

No Data in Dashboards

  • Check Grafana is running: docker compose ps
  • Verify database has recent data: sqlite3 website_analytics.db "SELECT MAX(Date) FROM user_data;"
  • Test queries: python3 scripts/test-queries.py
  • Check datasource UID matches: website-analytics-sqlite

API Authentication Errors

  • Verify SAS.json service account file exists
  • Check Google API quotas in Cloud Console
  • Ensure service account has access to GA4 property and Search Console

Grafana Login Issues

# Reset admin password
docker compose exec grafana grafana cli admin reset-admin-password newpassword

# Restart to clear lockouts
docker compose restart grafana

Analytics Advisor Agent

The plugin includes a specialised analytics-advisor agent that proactively assists with diagnostics, data interpretation, and optimisation recommendations. It activates automatically when you mention analytics issues, dashboard problems, SEO questions, or campaign performance.

Interpret Data

Explains metrics, identifies trends and anomalies, compares periods, and segments analysis.

Troubleshoot Infrastructure

SEO improvements, content strategy, campaign optimisation, and data quality fixes.
