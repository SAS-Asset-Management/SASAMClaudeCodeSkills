# Standard layer — generated locally per engagement

This pack does NOT ship the chunked MDR standard. The governing document is
licensed content; the rubrics, equations, and lookup tables in this pack are
SAS methodology, but the standard itself is not redistributable.

## Generating the chunks

Per engagement, chunk the licensed MDR document into the engagement repo's
local pack overlay, which the pack resolution order then prefers over this
directory:

```bash
python3 ${CLAUDE_PLUGIN_ROOT}/engine/chunker.py \
    --pdf <path to the licensed MDR pdf> \
    --manifest <chunkManifest.yaml> \
    --out <engagementRepo>/packs/mdr-governance-v3/standard/
```

The chunker emits `NNN_chunk.md` files plus an `INDEX.md` topic router. Do not
author `INDEX.md` by hand — it is generated by the chunker.

## Chunk manifest skeleton

The manifest is a list of `{file, title, startPage, endPage}` rows. The rows
below are illustrative placeholders only — the page numbers are NOT real and
must be replaced with the pagination of the licensed document in hand:

```yaml
# PLACEHOLDER example — replace pages with the licensed document's pagination
- file: 001_chunk.md
  title: "Introduction and definitions"
  startPage: 1
  endPage: 5
- file: 002_chunk.md
  title: "Condition rating methodology"
  startPage: 6
  endPage: 10
- file: 003_chunk.md
  title: "Data templates"
  startPage: 11
  endPage: 15
```

Aim for topic coherent chunks of roughly five pages each so a citing agent
loads exactly one chunk per clause, never the whole standard.
