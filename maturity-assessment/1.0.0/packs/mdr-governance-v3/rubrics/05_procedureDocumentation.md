# 05 — Procedure Documentation

## Definition

The extent to which condition assessment procedures are formally documented, version controlled, current, aligned to the MDR standard, and accessible to assessors. Covers procedure manuals, work instructions, inspection task cards, and inspection and test plans across all in scope asset subclasses.

## Evidence Types

- Named procedures, work instructions, inspection and test plans, or task cards
- Document control metadata including revision, issue date, owner, and review date
- A procedure register or index
- Explicit mapping from procedures to MDR standard requirements
- Asset subclass specific condition assessment manuals

## Maturity Ratings

| Level | Rubric |
| --- | --- |
| 0 | No written procedures exist and assessments are performed from tacit knowledge. |
| 1 | Unofficial procedures are scattered, with no document control and no alignment to the MDR standard. |
| 2 | Procedures exist for some asset classes but are applied inconsistently and are not explicitly aligned to the MDR standard, with methodology spread across many generic documents. |
| 3 | Every in scope assessment has a documented, controlled procedure that cites the MDR standard requirement, and a register is maintained. |
| 4 | Procedures are reviewed on cycle, feedback from assessors is incorporated, and usage analytics are captured. |
| 5 | Procedures are the single source of truth, integrated with digital workflow, and continuously improved from outcome data and external benchmarking. |

## Scoring Notes

- Cross references 06_assessorCompetency through 09_qaProcesses — if procedures do not exist, competency, rating calibration, evidence, and QA are all impacted downstream.
- Key artefact types: procedure register, work instruction set, technical maintenance plans, process architecture document.
- Methodology spread across many generic work instructions without a standalone MDR aligned procedure set caps the score at Level 2.
