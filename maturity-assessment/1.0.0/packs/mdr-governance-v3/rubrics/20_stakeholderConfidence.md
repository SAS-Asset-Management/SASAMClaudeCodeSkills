# 20 — Stakeholder Confidence

## Definition

The extent to which key stakeholders — the receiving authority, engineering, operations, maintenance, and finance — trust the condition assessment outputs and use them in their decisions. Covers declared confidence, observed use in forums, and the willingness of each function to commit resources on the basis of assessment findings.

## Evidence Types

- Meeting minutes, forums, or governance records citing assessment outputs
- Stakeholder surveys or feedback on the assessment framework
- Briefings prepared for decision making bodies
- Written concurrence with assessment findings from engineering or finance
- Patterns of use visible in investment and maintenance decisions

## Maturity Ratings

| Level | Rubric |
| --- | --- |
| 0 | Stakeholders are unaware of the assessment framework or do not use it. |
| 1 | Some stakeholders reference the outputs and material confidence concerns exist. |
| 2 | Confidence is mixed, with some functions using the outputs and others working around them. |
| 3 | All key stakeholder groups use assessment outputs in their formal decisions and confidence is declared and recorded. |
| 4 | A stakeholder feedback loop drives framework improvements and confidence trends are tracked. |
| 5 | The assessment framework is the authoritative condition view across the network and external parties rely on it without reservation. |

## Scoring Notes

- Cross references 14_uncertainty — undeclared uncertainty damages confidence when it surfaces — and all Domain 3 data quality subjects.
- Key artefact types: briefing packs to the receiving authority, stakeholder response registers, governance minutes.
- Polite acknowledgement in a meeting is not confidence; probe whether stakeholders commit resources on the basis of the outputs.
