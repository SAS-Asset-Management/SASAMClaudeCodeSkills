# 17 — Prioritisation Method

## Definition

The transparency, consistency, and analytical rigour of the method used to prioritise defects and renewals from condition assessment data. Covers the scoring or ranking logic, weightings, criticality inputs, and the auditable translation of raw condition records into a prioritised list of actions.

## Evidence Types

- A documented prioritisation algorithm, scoring matrix, or ranking rule set
- Criticality or consequence inputs such as safety, service, and cost
- Worked examples applying the method to real records
- Version control on the method with change history
- Outputs showing the prioritised list with reasoning per item

## Maturity Ratings

| Level | Rubric |
| --- | --- |
| 0 | No prioritisation method exists and actions are taken first come first served or ad hoc. |
| 1 | Prioritisation is informal, performed by subject matter experts, and is not reproducible. |
| 2 | A partial method exists for some asset classes with inconsistent inputs and no end to end auditability. |
| 3 | A documented prioritisation method is applied across all in scope asset classes with inputs traced to MDR records. |
| 4 | The method is tuned against outcome data with sensitivity analysis on weightings. |
| 5 | Prioritisation is probabilistic and risk based and is integrated with investment planning tooling. |

## Scoring Notes

- Cross references 16_investmentLinkage, 18_thresholdDefinition, and 19_renewalAlignment.
- Key artefact types: analysis scripts, gap assessment registers, technical maintenance plans.
- A prioritised list produced in spreadsheet form without documented rules is not a method, it is an opinion.
