# 23 — Assessment Frequency

## Definition

Whether each asset class is physically assessed at an interval appropriate to its criticality, deterioration rate, and failure consequence. Distinct from 04_reportingFrequency, which is about data submission cadence — this subject is the cadence of the inspections themselves.

## Evidence Types

- A defined assessment interval per asset class
- Justification for the interval based on criticality, deterioration rate, or standard
- Assessment schedule or plan
- Records of actual assessment dates against plan
- Review record where the interval is reconsidered against outcome data

## Maturity Ratings

| Level | Rubric |
| --- | --- |
| 0 | No frequency is defined and assessments are reactive. |
| 1 | A frequency exists informally for some assets with no schedule. |
| 2 | A frequency is defined for core asset classes but adherence is patchy and justification is thin. |
| 3 | Frequency is defined, justified, and adhered to across all in scope asset classes. |
| 4 | Frequency is tuned against observed deterioration and failure data with risk based variation applied. |
| 5 | Frequency is condition informed and dynamic, continuously optimised, and integrated with maintenance planning. |

## Scoring Notes

- Cross references 04_reportingFrequency and 24_dataCurrency — infrequent assessment guarantees stale data regardless of submission cadence.
- Key artefact types: inspection schedules, technical maintenance plans, capture frequency forecasts.
- A plan showing the right frequency is not evidence of adherence; compare planned to actual assessment dates on a sample of assets.
