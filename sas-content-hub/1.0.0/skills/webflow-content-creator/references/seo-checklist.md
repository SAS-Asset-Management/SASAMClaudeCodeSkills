# SEO Checklist

Use this checklist before publishing any content to ensure SEO optimisation.

---

## Title Tag (max 60 characters)

- [ ] Includes primary keyword
- [ ] Under 60 characters (including spaces)
- [ ] Compelling — would you click it in search results?
- [ ] Includes brand (SAS-AM) at end
- [ ] Unique — not duplicating another page

**Format:** `[Primary Keyword] [Supporting Context] | SAS-AM`

**Examples:**
- "AI Readiness for Water Utilities | SAS-AM" (42 chars)
- "Predictive Maintenance in Mining | Case Study | SAS-AM" (55 chars)
- "ISO 55001 Data Quality Requirements | SAS-AM" (46 chars)

---

## Meta Description (max 160 characters)

- [ ] Summarises key value proposition
- [ ] Includes primary keyword naturally
- [ ] Under 160 characters
- [ ] Includes call to action or benefit
- [ ] Compelling — would you click it?

**Format:** `[Value proposition]. [Benefit or CTA].`

**Examples:**
- "Learn the 5 steps water utilities need to become AI-ready. Practical guidance from SAS-AM's asset management experts." (119 chars)
- "How a major gold producer reduced unplanned downtime by 23%. Read the full case study." (87 chars)

---

## Heading Structure

- [ ] One H1 per page (the main title)
- [ ] H1 includes primary keyword
- [ ] H2s for major sections (3-5 per article)
- [ ] H3s for subsections within H2s
- [ ] Logical hierarchy (never skip levels)
- [ ] Headers describe content that follows

**Good structure:**
```
H1: Why Your AI Pilot Failed: Data Quality is the Hidden Killer
  H2: The Four Data Quality Killers
    H3: Inconsistent CMMS Records
    H3: Disconnected Sensor Data
  H2: What Actually Works
  H2: The Honest Answer
```

---

## Keyword Usage

- [ ] Primary keyword in H1
- [ ] Primary keyword in first 100 words
- [ ] Secondary keywords in H2s naturally
- [ ] Keywords used naturally — no stuffing
- [ ] Related terms and synonyms throughout
- [ ] Keyword density 1-2% (natural, not forced)

**Primary Keywords by Theme:**
- AI/ML: "predictive maintenance", "AI asset management", "machine learning maintenance"
- Reliability: "reliability engineering", "RCM", "FMEA", "root cause analysis"
- Maturity: "ISO 55001", "asset management maturity", "GFMAM"
- Data: "data quality", "AI-ready data", "CMMS data"

---

## Image Optimisation

### Hero Image

- [ ] 16:9 aspect ratio
- [ ] 1000px wide, under 200KB (via Squoosh)
- [ ] Descriptive filename (not IMG_1234.jpg)
- [ ] Alt text describes image content
- [ ] Alt text under 125 characters
- [ ] Alt text includes keyword if natural

**Good alt text:**
- "Maintenance engineer analysing predictive analytics dashboard on tablet"
- "Water treatment facility control room with digital monitoring systems"

**Bad alt text:**
- "Image"
- "Photo of engineer"
- "AI predictive maintenance asset management SAS-AM" (keyword stuffed)

---

## Internal Linking

- [ ] 2-4 internal links per article
- [ ] Link to related SAS-AM content
- [ ] Use descriptive anchor text
- [ ] Links are relevant to context
- [ ] Mix of recent and evergreen content

**Good anchor text:**
- "Learn more about predictive maintenance implementation"
- "See our ISO 55001 assessment approach"

**Bad anchor text:**
- "Click here"
- "Read more"
- "This article"

---

## URL/Slug

- [ ] URL-friendly (lowercase, hyphens)
- [ ] Under 75 characters
- [ ] Includes primary keyword
- [ ] No stop words (a, the, and, is)
- [ ] No special characters

**Good slugs:**
- `ai-readiness-water-utilities`
- `predictive-maintenance-mining-case-study`
- `iso-55001-data-quality`

**Bad slugs:**
- `Why-Your-AI-Pilot-Failed-Data-Quality-is-the-Hidden-Killer`
- `article-march-2026`
- `new_post_1`

---

## Content Quality (SEO Impact)

- [ ] Minimum 800 words for articles
- [ ] Answers user search intent
- [ ] Provides unique value (not just repackaging)
- [ ] Recent/updated content
- [ ] No duplicate content
- [ ] Scannable (short paragraphs, bullet points)
- [ ] Mobile-friendly formatting

---

## Technical SEO

- [ ] Publish date set correctly
- [ ] Author attribution (SAS-AM)
- [ ] Featured image uploaded
- [ ] All required CMS fields populated
- [ ] No broken internal links
- [ ] Page loads in under 3 seconds
