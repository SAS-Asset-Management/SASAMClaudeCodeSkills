## Using This in Proposals

When writing government tender responses or proposals, use PRINCE2 language to describe the marcov.GATE approach:

> "Our delivery methodology follows a stage-gated approach aligned with PRINCE2 principles. Each project progresses through defined stages (Initiation, Specialist Stages 1 and 2, Final Delivery, and Closing) with formal End Stage Assessments at each gate. Project governance is maintained through regular Highlight Reports and a structured Exception Management process."

Replace "marcov.GATE" with "our stage-gated delivery framework" in client-facing documents unless the client has specifically engaged with the marcov brand.

## DTF Gateway Alignment (Victorian Government)

The Victorian Government's Department of Treasury and Finance (DTF) uses Gateway Reviews at key decision points. marcov.GATE phases map to these:

| DTF Gateway | marcov.GATE Alignment |
|-------------|----------------------|
| Gateway 1: Strategic Assessment | Pre-project (BEAM sales phase) |
| Gateway 2: Business Case | Inception gate |
| Gateway 3: Readiness for Market | Discovery gate (requirements/specification complete) |
| Gateway 4: Tender Decision | N/A (SAS-AM is the supplier, not the procurer) |
| Gateway 5: Readiness for Service | Delivery gate |
| Gateway 6: Benefits Realisation | Close gate (benefits realisation plan) |
