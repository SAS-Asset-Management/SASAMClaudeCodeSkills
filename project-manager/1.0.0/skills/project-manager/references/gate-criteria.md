# marcov.GATE — Gate Criteria Reference

Evidence requirements for each gate, scaled by project tier. The skill acts as the gatekeeper — users do not self-certify.

---

## Gate Assessment Verdicts

| Verdict | Meaning | Action |
|---------|---------|--------|
| **ADVANCE** | All criteria met with sufficient evidence | Progress to next phase |
| **HOLD** | Some criteria partially met, gaps identified | Remain in current phase, address gaps |
| **ESCALATE** | Critical blocker preventing progress | Flag to project sponsor, define mitigation path |

---

## Gate 1: Inception → Discovery

### Micro Tier (Lightweight)
| Criterion | Evidence Required |
|-----------|-----------------|
| Scope agreed | Written confirmation from client (email or signed document) |
| Key contact identified | Name and contact details of primary client stakeholder |
| Timeline confirmed | Start and end dates agreed |

### Standard Tier
| Criterion | Evidence Required |
|-----------|-----------------|
| Scope document approved | Signed or email-approved scope document covering objectives, deliverables, exclusions |
| Stakeholders mapped | List of stakeholders with roles, influence levels, and contact details |
| Project plan agreed | Timeline with milestones, deliverable dates, and invoicing schedule |
| Team mobilised | Team members assigned with allocation percentages confirmed |
| Initial risks assessed | At least 3 risks identified with likelihood, impact, and mitigation strategies |

### Major Tier (Formal)
All Standard criteria PLUS:
| Criterion | Evidence Required |
|-----------|-----------------|
| Project Board constituted | Sponsor, senior user, and senior supplier identified (PRINCE2 overlay) |
| Communication plan documented | Frequency, format, and audience for status reporting agreed |
| Quality criteria defined | Acceptance criteria for each deliverable documented |
| Change control process agreed | How out-of-scope requests will be managed |
| Kick-off meeting completed | Minutes or notes from formal kick-off with client |

---

## Gate 2: Discovery → Development

### Micro Tier (Lightweight)
| Criterion | Evidence Required |
|-----------|-----------------|
| Data/information gathered | Key inputs collected from client |
| Approach confirmed | High-level method validated with client |

### Standard Tier
| Criterion | Evidence Required |
|-----------|-----------------|
| Data collected and validated | Data sources identified, data received, quality assessed |
| Gaps/requirements documented | Written gap analysis or requirements specification |
| Findings validated by client | Client confirmation that findings are accurate (meeting notes or email) |
| Approach/architecture confirmed | Technical approach or methodology documented and agreed |
| Discovery report delivered | Formal or informal report summarising findings |

### Major Tier (Formal)
All Standard criteria PLUS:
| Criterion | Evidence Required |
|-----------|-----------------|
| Requirements traceability established | Each requirement traceable to a business need |
| Architecture review completed | Peer review of technical design (software projects) |
| Updated risk register | Discovery findings reflected in updated risk assessment |
| Revised plan if needed | Any scope/timeline changes documented and approved |

---

## Gate 3: Development → Delivery

### Micro Tier (Lightweight)
| Criterion | Evidence Required |
|-----------|-----------------|
| Draft output reviewed | Client has seen and commented on the draft deliverable |
| Feedback addressed | Key feedback items incorporated |

### Standard Tier
| Criterion | Evidence Required |
|-----------|-----------------|
| Draft deliverables reviewed by client | Evidence of client review (marked-up document, meeting notes, email feedback) |
| Feedback incorporated | List of feedback items and how each was addressed |
| Internal quality review completed | Peer review or QA check performed by someone other than the author |
| Working software demonstrated | Client has seen working system with real or representative data (software projects) |
| No critical risks unmitigated | All high-likelihood/high-impact risks have active mitigation plans |

### Major Tier (Formal)
All Standard criteria PLUS:
| Criterion | Evidence Required |
|-----------|-----------------|
| User acceptance testing completed | UAT sign-off or test results documented (software projects) |
| Training materials prepared | User guides, training decks, or quick-start documentation |
| Deployment plan documented | Rollout plan with rollback procedure (software projects) |
| Updated risk register reviewed | Project Board has reviewed current risk position |

---

## Gate 4: Delivery → Close

### Micro Tier (Lightweight)
| Criterion | Evidence Required |
|-----------|-----------------|
| Final deliverable accepted | Written confirmation from client |
| Invoice submitted | Invoice sent |

### Standard Tier
| Criterion | Evidence Required |
|-----------|-----------------|
| Final deliverables accepted by client | Formal or email acceptance of all deliverables |
| Final presentation delivered | Presentation of findings/outcomes to client (advisory projects) |
| User training completed | Training sessions delivered with attendance recorded (software projects) |
| Deployment verified | System live and functioning in production (software projects) |
| Final invoice submitted | Invoice sent for remaining amount |

### Major Tier (Formal)
All Standard criteria PLUS:
| Criterion | Evidence Required |
|-----------|-----------------|
| Benefits realisation plan agreed | How the client will measure outcomes post-project |
| Support arrangements confirmed | Warranty period, support contact, escalation path documented |
| Formal acceptance signed | Client sign-off document executed |

---

## Gate 5: Close (Project Exit)

### All Tiers
| Criterion | Evidence Required |
|-----------|-----------------|
| Lessons learned captured | At least 3 lessons documented (what went well, what to improve) |
| Client feedback collected | NPS score, satisfaction survey, or informal feedback noted |
| Handover documentation complete | All project artefacts organised and accessible |
| Next opportunity identified | Follow-on work scoped or "none identified" recorded |
| Project archived | Repository tagged, status set to completed |
