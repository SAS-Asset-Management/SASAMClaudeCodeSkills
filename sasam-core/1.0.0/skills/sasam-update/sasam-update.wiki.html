SASAM Update Skill v1.0.0

A Claude Code skill for managing updates to the SASAMClaudeCodeSkills marketplace. Check for new versions, view changelogs, back up local modifications, and apply updates — all from within a Claude Code conversation.

Check Updates

Compare your local VERSION file against the remote GitHub repository to detect available updates.

View Changelog

Fetch and display the remote CHANGELOG to review what changed before applying an update.

Apply Updates

Back up local modifications, pull latest changes from GitHub, and re-register all skills automatically.

Version Info

Display the current installed version alongside a list of all registered skills and their versions.

Safe by default — the update skill always backs up locally modified files before pulling changes. If a conflict occurs, you will be notified and no data is lost.

How It Works

When you invoke the SASAM Update skill, it follows a structured workflow to ensure safe, reliable updates:

  1. Read local VERSION file — determine the currently installed version from VERSION at the repository root
  2. Fetch remote VERSION from GitHub — retrieve the latest published version from the raw GitHub URL
  3. Compare versions — determine whether an update is available, and whether it is a major, minor, or patch release
  4. Display changelog — fetch and present the remote CHANGELOG.md so you can review changes before proceeding
  5. Backup local modifications — identify and archive any locally modified files to prevent data loss
  6. Pull updates and re-register — execute git pull, run register-commands.sh, and regenerate the marketplace manifest

Check for Updates

The check workflow compares your local installation against the remote repository in three steps:

Step 1: Read Local Version

The skill reads the VERSION file at the root of your SASAMClaudeCodeSkills clone:

# Read the local version
LOCAL_VERSION=$(cat "$SASAM_ROOT/VERSION")
echo "Local version: $LOCAL_VERSION"

Step 2: Fetch Remote Version

The latest published version is fetched from GitHub using the raw content URL:

# Fetch remote version from GitHub
REMOTE_VERSION=$(curl -s \
  "https://raw.githubusercontent.com/marcov-workshop/SASAMClaudeCodeSkills/main/VERSION")
echo "Remote version: $REMOTE_VERSION"

Step 3: Compare Versions

A semantic version comparison determines whether an update is available and its severity:

# Compare local vs remote
if [ "$LOCAL_VERSION" = "$REMOTE_VERSION" ]; then
  echo "You are up to date (v$LOCAL_VERSION)"
elif [ "$(printf '%s\n' "$LOCAL_VERSION" "$REMOTE_VERSION" | sort -V | head -1)" = "$LOCAL_VERSION" ]; then
  echo "Update available: v$LOCAL_VERSION -> v$REMOTE_VERSION"
else
  echo "Local version is ahead of remote"
fi

Tip: Run /sasam-update check periodically to stay informed about new features, bug fixes, and skill additions.

View Changelog

Before applying an update, you can review the changelog to understand what has changed:

# Fetch the remote changelog
CHANGELOG=$(curl -s \
  "https://raw.githubusercontent.com/marcov-workshop/SASAMClaudeCodeSkills/main/CHANGELOG.md")

# Display the changelog
echo "$CHANGELOG"

The changelog follows Keep a Changelog format and is organised by version, with entries grouped under Added, Changed, Fixed, and Removed headings.

Apply Updates

The apply workflow is the most comprehensive operation. It performs seven steps to safely update your installation:

Step 1: Check for Local Modifications

Scan the working tree for any locally modified files that might conflict with the update:

# Detect locally modified files
MODIFIED=$(git -C "$SASAM_ROOT" diff --name-only)
if [ -n "$MODIFIED" ]; then
  echo "Local modifications detected:"
  echo "$MODIFIED"
fi

Step 2: Backup Modified Files

Create a timestamped backup of any locally modified files before pulling:

# Create backup directory
BACKUP_DIR="$SASAM_ROOT/.sasam-backups/$(date +%Y%m%d-%H%M%S)"
mkdir -p "$BACKUP_DIR"

# Copy modified files to backup
echo "$MODIFIED" | while read -r file; do
  mkdir -p "$BACKUP_DIR/$(dirname "$file")"
  cp "$SASAM_ROOT/$file" "$BACKUP_DIR/$file"
done
echo "Backup saved to: $BACKUP_DIR"

Step 3: Pull Latest Changes

Fetch and merge the latest changes from the remote repository:

# Pull latest from origin/main
git -C "$SASAM_ROOT" pull origin main

Step 4: Re-register Skills

Run the registration script to update all skill symlinks and Claude Code configuration:

# Re-register all skills
bash "$SASAM_ROOT/register-commands.sh"

Step 5: Update Marketplace Clone

Ensure the local marketplace directory reflects the latest skill catalogue:

# Sync marketplace metadata
rsync -a --delete "$SASAM_ROOT/" "$MARKETPLACE_DIR/" \
  --exclude='.git' --exclude='.sasam-backups'

Step 6: Regenerate Manifest

Rebuild the sasam-file-manifest.json with updated SHA-256 hashes for all tracked files:

# Regenerate file manifest with SHA-256 hashes
find "$SASAM_ROOT" -type f \
  -not -path '*/.git/*' \
  -not -path '*/.sasam-backups/*' \
  -exec sha256sum {} \; | sort > "$SASAM_ROOT/sasam-file-manifest.json"

Step 7: Display Success

# Confirm update
NEW_VERSION=$(cat "$SASAM_ROOT/VERSION")
echo "Successfully updated to v$NEW_VERSION"
echo "All skills re-registered and manifest regenerated."

Warning: If a git merge conflict occurs during Step 3, the update will pause and display the conflicting files. Resolve conflicts manually, then run /sasam-update apply again.

Manifest Verification

The skill uses sasam-file-manifest.json to verify file integrity after updates. Each entry contains a file path and its expected SHA-256 hash.

How Verification Works

After pulling updates, the skill iterates over every entry in the manifest and compares the computed hash against the expected value:

# Verify file integrity against manifest
MANIFEST="$SASAM_ROOT/sasam-file-manifest.json"
ERRORS=0

while IFS= read -r entry; do
  FILE_PATH=$(echo "$entry" | jq -r '.path')
  EXPECTED_HASH=$(echo "$entry" | jq -r '.sha256')

  if [ -f "$SASAM_ROOT/$FILE_PATH" ]; then
    ACTUAL_HASH=$(sha256sum "$SASAM_ROOT/$FILE_PATH" | awk '{print $1}')
    if [ "$ACTUAL_HASH" != "$EXPECTED_HASH" ]; then
      echo "MISMATCH: $FILE_PATH"
      echo "  Expected: $EXPECTED_HASH"
      echo "  Actual:   $ACTUAL_HASH"
      ERRORS=$((ERRORS + 1))
    fi
  else
    echo "MISSING: $FILE_PATH"
    ERRORS=$((ERRORS + 1))
  fi
done < <(jq -c '.files[]' "$MANIFEST")

if [ "$ERRORS" -eq 0 ]; then
  echo "All files verified successfully."
else
  echo "$ERRORS file(s) failed verification."
fi

Integrity guarantee: If any file fails hash verification, the skill reports the mismatch immediately. This catches corrupted downloads, incomplete transfers, and unexpected local changes.

Auto-Registration

Skills are automatically registered via register-commands.sh, which is triggered both manually and by git hooks.

register-commands.sh

This script scans the repository for skill directories, creates the necessary symlinks, and updates Claude Code's configuration to recognise each skill:

# register-commands.sh scans for SKILL.md files
# and registers each skill with Claude Code
find "$SASAM_ROOT" -name "SKILL.md" -type f | while read -r skill_file; do
  SKILL_DIR=$(dirname "$skill_file")
  SKILL_NAME=$(basename "$SKILL_DIR")
  echo "Registering: $SKILL_NAME"
  # Create symlink and update config...
done

Git Hooks

Two git hooks ensure skills are re-registered automatically whenever the repository changes:

HookTriggerAction
post-checkoutAfter git checkout or git switchRuns register-commands.sh to sync skills with the checked-out branch
post-mergeAfter git pull or git mergeRuns register-commands.sh to register any new or updated skills

No manual steps required: After git pull, the post-merge hook automatically re-registers all skills. The /sasam-update apply command also calls register-commands.sh explicitly as a safety net.

Error Handling

The update skill handles common failure scenarios gracefully and provides clear error messages:

Network Error

If the remote GitHub URL is unreachable (no internet, DNS failure, or repository moved):

ERROR: Unable to fetch remote VERSION from GitHub.
  URL: https://raw.githubusercontent.com/marcov-workshop/SASAMClaudeCodeSkills/main/VERSION
  Check your network connection and try again.
  If the problem persists, verify the repository URL has not changed.

Git Conflict

If git pull encounters merge conflicts during the apply workflow:

WARNING: Merge conflict detected in the following files:
  - sasam-core/1.0.0/skills/sasam-update/SKILL.md
  - register-commands.sh

Your local modifications have been backed up to:
  .sasam-backups/20260309-143022/

Resolve the conflicts manually, then run:
  /sasam-update apply
