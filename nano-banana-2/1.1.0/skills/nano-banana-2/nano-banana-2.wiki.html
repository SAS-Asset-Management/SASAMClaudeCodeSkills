Nano Banana 2 v1.1.0

Generate images from text using Google's Gemini API with web search grounding. Create production-ready visuals with automatic SAS watermarking and post-processing.

Text-to-Image

Generate images from natural language descriptions using Gemini's multimodal capabilities.

Web Grounding

Leverage web search grounding for more accurate and contextually relevant image generation.

Post-Processing

Automatic SAS watermark overlay, JPG optimisation, and size targeting for every generated image.

Multi-Size

Support for multiple aspect ratios and output sizes from 512px thumbnails to 4K exports.

How It Works

Nano Banana 2 follows a five-step pipeline from prompt to final deliverable:

1. Check Configuration

Verify the Gemini API key is configured and the skill environment is ready.

2. Parse Prompt

Interpret the user's text prompt, extract any size/aspect ratio modifiers, and prepare the generation request.

3. Call Gemini API

Send the prompt to the gemini-3.1-flash-image-preview model with web search grounding enabled.

4. Save Raw PNG

Write the raw generated image to disk as a PNG file in the working directory.

5. Post-Process & Deliver

Apply the SAS watermark, optimise to JPG (target 1000px, <200KB), and deliver the final image to the user.

Image Generation

Image generation is powered by Google's Gemini API. The skill uses the gemini-3.1-flash-image-preview model, which supports multimodal text-to-image generation with high quality results.

Web Search Grounding

Web search grounding is enabled by default. This means the model can search the web during generation to produce more accurate, up-to-date, and contextually relevant images. For example, if you ask for a logo of a real company or a landmark, grounding helps the model reference actual visual information.

How grounding works: When enabled, the Gemini API performs a web search related to your prompt before generating the image. This provides the model with real-world visual context, resulting in more accurate depictions of real subjects.

Generation Pipeline

The API returns a raw PNG image which is saved locally. This raw output must then go through the mandatory post-processing step (see Mandatory Post-Processing) before being delivered to the user.

Prompt Engineering

Effective prompts lead to better image results. Here are guidelines for crafting prompts that work well with Nano Banana 2.

Tips for Effective Prompts

  • Be specific about the subject, setting, and visual style
  • Include lighting and mood descriptors (e.g. "warm golden hour lighting")
  • Mention the perspective or camera angle if relevant
  • Specify artistic style: photorealistic, illustration, watercolour, isometric, etc.
  • Use adjectives for quality: "high detail", "professional", "clean background"
  • Reference colour palettes when brand consistency matters

What Works Well

Good Prompt

"A professional isometric illustration of a modern office building with glass facades, surrounded by green trees, clean white background, corporate style"

Good Prompt

"A photorealistic close-up of a circuit board with glowing blue traces, shallow depth of field, dark background, tech aesthetic"

Common Pitfalls

Avoid these mistakes:

Do not include text like "add a SAS watermark" in your prompt — the watermark is applied automatically during post-processing. Do not request specific file formats in the prompt. Avoid overly vague prompts like "something nice" — be descriptive.

Mandatory Post-Processing

CRITICAL: Every generated image MUST be post-processed before delivery.

Raw PNG files from the Gemini API must never be delivered directly to the user. All images must pass through the post-processing script to add the SAS watermark and optimise the output.

Post-Processing Command

node ~/.claude/SASAMClaudeCodeSkills/nano-banana-2/1.1.0/scripts/post-process.js <input.png> <output_final.jpg>

What Post-Processing Does

Step Description
SAS Logo Watermark Overlays the SAS logo watermark on the generated image
JPG Optimisation Converts the raw PNG to an optimised JPG format
Size Targeting Resizes to target 1000px on the longest edge, <200KB file size

DO NOT add watermark text to the generation prompt — the watermark is applied programmatically during post-processing.

DO NOT deliver the raw PNG file to the user. Always deliver the post-processed _final.jpg output.

Setup

Before using Nano Banana 2, you need to configure your Gemini API key.

Getting a Gemini API Key

  1. Visit Google AI Studio at https://aistudio.google.com/
  2. Sign in with your Google account
  3. Navigate to API Keys and create a new key
  4. Copy the API key for use in setup

First-Time Configuration

Run the setup command to store your API key:

/nano-banana-2 setup

The API key is stored at:

~/.claude/skills/nano-banana-2/config.json

Security note: The config file is stored in your local ~/.claude/skills/ directory and is not committed to version control. Keep your API key confidential.

Config File

The configuration file is a JSON document stored at ~/.claude/skills/nano-banana-2/config.json.

Config Structure

{
  "geminiApiKey": "YOUR_GEMINI_API_KEY",
  "model": "gemini-3.1-flash-image-preview",
  "webGrounding": true,
  "defaultSize": "1K",
  "defaultAspectRatio": "1:1",
  "outputFormat": "jpg",
  "postProcess": {
    "addWatermark": true,
    "targetWidth": 1000,
    "maxFileSize": "200KB"
  }
}
Field Type Description
geminiApiKey string Your Google Gemini API key (required)
model string Gemini model identifier for image generation
webGrounding boolean Enable web search grounding for generation
defaultSize string Default output size (512px, 1K, 2K, 4K)
defaultAspectRatio string Default aspect ratio for generated images
outputFormat string Final output format after post-processing
postProcess object Post-processing configuration block

Error Handling

Common errors and how to resolve them:

API Key Missing

If you see "API key not configured", run /nano-banana-2 setup to enter your Gemini API key. The key must be stored in ~/.claude/skills/nano-banana-2/config.json before generation can proceed.

Rate Limit Exceeded

The Gemini API has rate limits. If you encounter a rate limit error, wait a few moments before retrying. Consider reducing the frequency of generation requests or upgrading your API tier for higher limits.

Generation Failed

If image generation fails, check the following: ensure your prompt does not violate content policies, verify your API key is valid and has sufficient quota, and confirm network connectivity. Try simplifying or rephrasing your prompt.
