SAS Presentation Skill v1.0.0

A Claude Code skill for creating professional, branded Reveal.js presentations following the marcov / SAS-AM communication style. Outputs standalone HTML + CSS with no build step — open in any browser, share via USB, or host on a static server.

17 Types

From pitch decks to meeting minutes — each with its own structure and tone.

Light / Dark

Built-in theme toggle with localStorage persistence. All colours via CSS variables.

24 Chart Types

Inline SVG chart library — theme-aware, accessible, interactive on hover.

WCAG 2.1 AA

Focus indicators, reduced motion, screen reader support, semantic HTML.

SAS-AM Tagline — every presentation MUST include on at least one slide:

"At SAS-AM we help organisations understand their strengths and weaknesses, model and predict the future, and make the right decisions through audit and benchmarking, advanced analytics and deep advisory services."

How It Works

When you ask Claude to create a presentation, the skill follows a structured flow:

  1. Discovery Interview — identify the type, gather content, plan charts
  2. Scaffold — create directory, copy base styles & logos
  3. Build Slides — title, sections, components, charts
  4. Review & Refine — test navigation, themes, run checklist
  5. Export (optional) — PDF via decktape

File Structure

presentation-folder/
├── presentation.html    # Main presentation file
├── styles.css           # Custom theme styles (copied from base-styles.css)
└── assets/              # Slide images (PNG, JPG, SVG)

SAS logos are loaded from Webflow CDN — no local logo files needed. Dual-theme images use the -blue / -green suffix convention and are switched via .logo-light / .logo-dark CSS classes.

Phase 2: Gather Content Details

5. Topic & Key Message

  • What is this about?
  • What is the single most important takeaway?
  • Is there a specific call-to-action?

6. Delivery Context

  • Where will this be presented? (conference, boardroom, webinar, shared via USB/email)
  • How much time do you have? (affects slide count)
  • Will you be presenting live or is it for self-navigation?

7. Branding

  • Use marcov / SAS-AM branding (default) or a client's brand?
  • If client branding, what are their brand colours?

8. Visual Assets

  • Do you have specific images, diagrams, or visualisations?
  • Should I create conceptual SVG diagrams?
  • Do you need a QR code for the closing slide?

9. Content Outline

  • Do you have existing content, bullet points, or a rough outline?
  • What are the key sections or topics to cover?
  • Review the section list for the selected type — add, remove, or reorder?

Phase 3: Data & Visualisation Planning

If the presentation includes data, metrics, or KPIs, walk through chart selection and data sourcing before building.

Step A: What Are You Trying to Show?

Use this decision tree to recommend chart types:

You want to show…Recommended Charts
Ranking — biggest / smallestHorizontal Bar, Treemap
Comparison — side by sideGrouped Bar, Radar / Spider
Composition — parts of a wholeStacked Bar, Donut, Treemap
Trend over timeLine, Area, Sparkline
Correlation — two variablesScatter, Bubble
Distribution — how data is spreadDistribution, Violin, Histogram
Flow / process — stage transitionsSankey, Funnel
Relationships — connectionsNetwork, Chord Diagram
Status / healthRAG Grid, Gauge, Progress Bars, Circular Bar
Strategic positioningQuadrant Chart
Cumulative changeWaterfall
Intensity (two dimensions)Heatmap / Matrix
Single KPI headlineGauge, Sparkline, Circular Bar

Step B: Where Is the Data?

Classify every data element into one of three source types:

Structured File

CSV, Excel, JSON, or database export. Real numbers exist in a file.

Ask: file path, columns, filters, aggregation method.

Label: "Source: FY24 Asset Register extract"

Anecdotal / Estimated

Rough numbers or proportions from experience — no file.

Ask: categories, rough %, relative sizes. Suggest splits and confirm.

Label: "Indicative — based on operational experience"

Mixed

Some series from files, some estimated. Handle each independently.

Ask: identify which is which. Footnote sourced vs estimated.

Label: cite each series separately.

Step C: Confirm Before Building

Always confirm data with the user:

"Here's what I'll put on the chart: [labels and values]. Does that look right?"

"Should I add a source line? Something like 'Source: FY24 Asset Register' or 'Indicative estimates, March 2026'?"

Chart Interactivity Pattern

  • Add class="chart-interactive" to the parent <svg> for sibling dimming on hover
  • Place <g class="chart-svg-tooltip"> after each data element for no-JS tooltip fallback
  • Use --chart-1 through --chart-6 for the categorical colour palette
  • Always include role="img" and aria-label on the SVG element

Structural Rules

  • Each slide is a <section> with a unique id and data-section
  • <h1> — title slide only (one per deck)
  • <h2> slide titles, <h3> card headings, <h4> sub-headings — never skip levels
  • All <img> must have alt text; decorative: alt="" role="presentation"
  • Inline SVGs preferred over image files
  • CSS variables for all colours — no hardcoded values
  • All JS at bottom of <body>
  • No build step — open-in-browser ready

Theme Management

Three states: light (default), dark, system (follows OS preference). Stored in localStorage under key marcov-presentation-theme.

The data-theme attribute is applied to <html>, <body>, and .reveal-viewport. Dual-theme images use:

<img src="assets/visual-blue.png" class="logo-light">
<img src="assets/visual-green.png" class="logo-dark">

Accessibility (WCAG 2.1 AA)

Colour Contrast

Normal text: 4.5:1 minimum
Large text (≥24px): 3:1
UI components: 3:1
Test both themes independently

Keyboard Access

All interactive elements focusable via Tab
Visible focus ring via --focus-ring
Custom buttons: role="button" + tabindex="0"

Screen Readers

lang="en-AU" on html
Descriptive alt on all images
aria-hidden="true" on decorative elements
.sr-only for hidden labels

Reduced Motion

Mandatory prefers-reduced-motion rule in every stylesheet. All animations to 0.01ms, single iteration.

Animations

Binary Background

Subtle falling binary code columns. Very low opacity: rgba(0,34,68,0.04) light / rgba(255,255,255,0.03) dark. Columns use aria-hidden="true".

QR Code Pulse

Closing slide QR glow: radial gradient with 2s ease-in-out pulse. Uses var(--accent).

SVG Data Flow

Use <animate> elements for conceptual diagrams (e.g. data flowing between nodes).

Pre-Delivery Checklist

HTML Compliance

  • <!DOCTYPE html> declared
  • lang="en-AU" set
  • UTF-8 charset
  • Viewport meta present
  • Description and author meta tags
  • Title with | marcov suffix
  • All slides have unique IDs
  • All slides have data-section attributes
  • Heading hierarchy correct (h1 → h2 → h3)
  • No hardcoded colours

Accessibility

  • All images have alt text
  • Colour contrast passes 4.5:1
  • Focus indicators visible
  • ARIA labels on icon-only buttons
  • Reduced motion rule in stylesheet
  • Keyboard navigation works
  • Binary columns have aria-hidden="true"

Functionality

  • Light/dark toggle works
  • Footer nav tracks correctly
  • Footer nav items match the type's sections
  • All fragments animate
  • Hash URLs resolve
  • Opens standalone in browser
  • Australian English spelling
  • PDF export tested (if needed)

Data Visualisations

  • Chart type appropriate for the data story
  • Data source identified (structured, anecdotal, or mixed)
  • Data values confirmed with user before building
  • All charts have role="img" and aria-label
  • All charts use .chart-interactive
  • Chart colours use --chart-1 through --chart-6
  • Tooltips present
  • Source attribution on every chart
  • Anecdotal data labelled as estimated/indicative
  • Axis labels, legends, values accurate and readable

Content

  • Discovery interview completed (Phases 1, 2, and 3)
  • Presentation type identified and confirmed
  • Section structure follows the type definition
  • Tone matches the type
  • SAS-AM tagline included on at least one slide
