The Problem

Asset-intensive organisations generate thousands of work orders every month. Almost none of them are correctly classified against a failure mode taxonomy — because doing so manually is too slow, too inconsistent, and too expensive at scale.

Without classification, maintenance data cannot drive reliability improvement. Failure patterns stay hidden. The same asset failures repeat. The FMEA taxonomy becomes a compliance artefact rather than a living operational tool.

2

The Solution

marcov.WOclassifier is a purpose-built AI classification engine that reads the full free text of each work order and maps it to the client's FMEA failure mode taxonomy. It returns structured, auditable results for every work order — including multi-attribute confidence scoring, primary and secondary work separation, and automatic FMEA gap detection.

1
Ingest Work order history provided as CSV, XLSX, or via direct API connection to IBM Maximo. FMEA taxonomy uploaded as a structured spreadsheet.
2
Classify The AI reasoning engine reads each work order's long text and maps it to the closest failure mode, returning per-attribute confidence scores (maintainable item, part, mechanism, cause, failure mode) alongside a written reasoning justification.
3
Export Structured results returned as CSV, XLSX, or via API back to Maximo. Each record includes: failure mode, maintainable item, part, mechanism, cause, confidence level, certainty status, and reasoning. FMEA gap report included.
3

Proven in the Field

marcov.WOclassifier has been validated against a real water utility CMMS dataset. 3,126 work orders were classified against a plant FMEA taxonomy in a single batch run, producing structured, auditable results across all classification categories.

  • Multi-attribute confidence scoring across five FMEA dimensions per classification
  • Automatic separation of primary work from secondary or opportunistic repairs
  • Automatic FMEA gap detection with suggested taxonomy additions
  • Fuzzy ID matching to correct taxonomy mismatches before output
  • Full written reasoning for every classification — defensible and reviewable
  • Three certainty tiers: High Confidence, Needs Review, and FMEA Gap
